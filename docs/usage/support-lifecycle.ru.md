Эта страница описывает, как пользователи получают помощь при эксплуатации Exordos Core, получают обновления и участвуют в развитии продукта.

## Каналы обращений

- [GitHub Issues](https://github.com/exordos/exordos_core/issues) — сообщения об ошибках, предложения по улучшению и запросы новых возможностей.
- [support@exordos.com](mailto:support@exordos.com) — консультации по установке, настройке и эксплуатации платформы, а также обращения, для которых публикация сведений в открытом issue недопустима.

При обращении укажите версию Exordos Core, используемую операционную систему, выполненную команду, полный текст ошибки и действия, после которых проблема возникает. Не передавайте пароли, закрытые ключи, токены и иные секреты.

## Техническая поддержка

Поддержка помогает пользователям установить и настроить платформу, настроить доступ, диагностировать проблемы и применять обновления. Самостоятельная диагностика начинается с [Руководства администратора](admin-guide.md) и [Диагностики проблем](troubleshooting.md).

Порядок обработки обращения:

1. Пользователь направляет обращение через GitHub Issues или электронную почту.
2. Обращение анализируется; при необходимости запрашиваются дополнительные сведения для воспроизведения проблемы.
3. Пользователь получает рекомендации по настройке, эксплуатации или устранению неисправности.
4. После устранения проблемы или предоставления ответа обращение закрывается.

## Устранение неисправностей

Для известных проблем используйте [Диагностику проблем](troubleshooting.md). Она содержит проверки состояний ресурсов, Node и гипервизоров, а также действия для распространённых ошибок конфигурации и bootstrap-скриптов.

Если проблема не описана в документации, создайте GitHub Issue или направьте обращение по электронной почте. После подтверждения дефекта исправление подготавливается в исходном коде проекта и включается в последующую версию Exordos Core.

## Обновления и исправления

Новые версии могут содержать исправления ошибок, улучшения безопасности и новые возможности. Доступные версии публикуются в [релизах Exordos Core](https://github.com/exordos/exordos_core/releases).

Перед обновлением:

1. Ознакомьтесь с изменениями целевой версии.
2. Сохраните версию установленного элемента, конфигурацию сети и учётные данные доступа.
3. Проверьте обновление на отдельной локальной инсталляции.
4. Подготовьте стратегию отката.

Подробные требования к обновлению приведены в [Руководстве администратора](admin-guide.md#обновление). Обновления выполняются поэлементно. Ядро Exordos Core также является элементом и обновляется стандартной командой:

```bash
exordos elements update core
```

## Совершенствование продукта

Пользователи могут предлагать улучшения, новые возможности и изменения поведения через GitHub Issues или по адресу [support@exordos.com](mailto:support@exordos.com). Предложения анализируются с учётом технической реализуемости, безопасности, совместимости и потребностей пользователей.

Принятые изменения реализуются в исходном коде, проходят проверку и включаются в будущие релизы. Сведения о выпущенных изменениях публикуются в материалах релиза.
