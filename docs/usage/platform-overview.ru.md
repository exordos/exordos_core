Exordos Core — open-source NoOps-платформа для автоматизации развёртывания, управления и эксплуатации IT-инфраструктуры и корпоративного программного обеспечения. Платформа применяется в локальных, частных, публичных и гибридных контурах.

Exordos Core выводит управление инфраструктурой за пределы классического DevOps-подхода. Вместо набора разрозненных инструментов, требующих постоянной ручной координации, платформа объединяет инфраструктуру, политики доступа и жизненный цикл сервисов в едином контуре управления. Это снижает операционные затраты, устраняет накопление конфигурационных расхождений и позволяет командам работать на уровне целей, а не отдельных низкоуровневых операций.

## Основные принципы

### Декларативное управление

Манифест описывает ресурсы, их параметры, зависимости, конфигурации и правила жизненного цикла. Платформа сама определяет операции, необходимые для достижения описанного состояния.

### Цикл согласования

Exordos Core постоянно сопоставляет фактическое состояние управляемых ресурсов с состоянием, описанным в манифесте. При обнаружении расхождения платформа выполняет необходимые действия для восстановления целевого состояния.

### Образ-ориентированное развёртывание

Вычислительные узлы создаются и обновляются на основе образов. При обновлении корневой диск Node заменяется содержимым целевого образа; подключённые дисковые тома и сетевые идентификаторы сохраняются. Этот подход обеспечивает воспроизводимость окружений и снижает конфигурационные расхождения между узлами.

### Elements

Программное обеспечение поставляется в виде элементов. Элемент содержит манифест с описанием зависимостей, инфраструктурных ресурсов, конфигураций, секретов и правил установки, обновления и удаления.

## Основные компоненты

- **Compute** — создание и управление виртуальными машинами и физическими серверами.
- **Storage** — создание и подключение дисковых томов.
- **Network** — управление сетевыми ресурсами.
- **DNS** — управление DNS-ресурсами.
- **Load Balancer** — управление балансировщиками нагрузки.
- **IAM** — управление пользователями, ролями и правами доступа.
- **Secrets** — хранение паролей, SSH- и RSA-ключей, сертификатов и других конфиденциальных данных.
- **Configs** — хранение и доставка конфигурационных данных программным компонентам.
- **Services** — управление системными сервисами на вычислительных узлах.
- **Values Store** — управление значениями, переменными и профилями для параметризации манифестов.
- **Element Manager** — установка, обновление и управление жизненным циклом элементов.
- **Marketplace** — публичные и приватные каталоги элементов.

## Основные ресурсы платформы

Платформа предоставляет и другие типы ресурсов; ниже перечислены основные из них.

| Ресурс | Назначение |
| --- | --- |
| Node | Виртуальная машина или физический сервер. Node не является контейнером. |
| Set | Набор однотипных Node с заданным количеством экземпляров и стратегией обновления. |
| Volume | Подключаемый дисковый том для хранения данных. |
| Network | Сетевой ресурс для связи вычислительных узлов и сервисов. |
| Config | Конфигурационные данные, доставляемые компонентам. |
| Secret | Конфиденциальные данные: пароли, ключи и сертификаты. |
| Service | Системный сервис, запускаемый на Node. |
| Element | Пакет прикладного или инфраструктурного программного обеспечения. |

## Функциональные возможности

Exordos Core обеспечивает:

- создание, настройку и удаление виртуальных машин и физических серверов;
- формирование наборов Node для масштабирования нагрузки;
- обновление Set по стратегиям `in-place`, `replace` и `rolling` с возможностью отката;
- управление дисковыми томами, сетевыми ресурсами, DNS и балансировщиками нагрузки;
- установку, обновление и удаление элементов с разрешением зависимостей;
- централизованное управление конфигурациями, секретами, значениями, переменными и профилями;
- управление учётными записями, ролями и правами доступа;
- создание, запуск, остановку и обновление системных сервисов;
- поиск, установку и распространение элементов через публичные и приватные каталоги;
- интеграцию с внешними системами через HTTP API сервисов платформы.
